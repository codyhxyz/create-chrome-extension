# Launch Materials

## Chrome Web Store Asset Requirements

| Asset | Size | Required? |
|---|---|---|
| Screenshots | 1280x800 or 640x400 | Yes (1-5) |
| Small promo tile | 440x280 | Optional |
| Marquee promo tile | 1400x560 | Optional |
| Extension icon | 128x128 | Yes (auto-generated) |

### Icon

The 128x128 store icon is auto-generated by `@wxt-dev/auto-icons` from the source icons in `public/icon/`. To customize:

1. Replace the PNGs in `public/icon/` with your own (or provide an SVG at `assets/icon.svg`)
2. Run `npm run build` -- the module generates all required sizes

### Screenshots

At least one screenshot is required. Up to 5 are allowed. Accepted sizes:
- **1280x800** (recommended)
- **640x400**

## Generating Screenshots

Use the `app-store-screenshots` skill in Claude Code to generate polished marketing screenshots programmatically.

### Tips for Good Screenshots

- **Show the extension in action** on a real page, not just the popup in isolation
- **Include both light and dark mode** screenshots if your extension supports both
- **Add annotations** (arrows, callouts, labels) highlighting key features
- **Use a clean browser window** -- hide bookmarks bar and other extensions
- **Capture the full context** -- show enough of the page to understand what the extension does
- **First screenshot matters most** -- it's the thumbnail in search results

### Manual Screenshot Process

1. `npm run build` and load the extension unpacked
2. Navigate to a representative page
3. Activate the extension (open popup, trigger content script, etc.)
4. Take a screenshot at exactly 1280x800 or 640x400
5. Annotate if needed (Figma, Canva, or any image editor)

## Promotional Tiles (Optional)

These are only used if Chrome Web Store features your extension:

- **Small promo tile (440x280):** Appears in CWS category pages
- **Marquee promo tile (1400x560):** Appears in CWS featured banners

Keep these simple: extension name, icon, one-line description, and a clean visual.
